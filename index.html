    Introduction

    This is the private API used by the Anova Precision Oven mobile applications.

    Authentication

    curl 'https://identitytoolkit.googleapis.com/v1/accounts:signInWithPassword?key=AIzaSyCGJwHXUhkNBdPkH3OAkjc9-3xMMjvanfU' \
-H 'Content-Type: application/json' \
--data-binary '{"email":"user@example.com","password":"PASSWORD","returnSecureToken":true}'

    {
   "idToken": "",
   "refreshToken": "",
   "expiresIn": "",
}

    Be sure to store the idToken as you'll need it to authenticate to the websocket. Also note that it's a JWT token, so you can extract additional information about the token if you're interested.

    Anova uses a Firebase application to authenticate its users with a password.

    POST https://identitytoolkit.googleapis.com/v1/accounts:signInWithPassword

    Query Parameters

    Parameter Required Description
    key true The Firebase application key

    Body Parameters

    Parameter Required Description
    email true The email of the Anova mobile application user
    password true The password for the Anova mobile application user
    returnSecureToken true Informs the API to return a JWT token

    Websocket Commands

    You must connect to wss://app.oven.anovaculinary.io.

    Authentication Command

    {
  "command": "AUTH_TOKEN",
  "payload": "[ID_TOKEN]",
}

    The service will respond with the following message

    {"response":"AUTH_TOKEN_RESPONSE"}

    In order to authenticate, you must send a message containing the idToken you received in the earlier authorization call as the payload.

    Send Command

    {
  "command": "SEND_OVEN_COMMAND",
  "payload": {
    "deviceId": "<DEVICE_ID>",
    "command": <COMMAND>,
    "requestId": "<REQUEST_UUID>"
  }
}

    The service will acknowledge the message with the following response

    {
  "response": "OVEN_COMMAND_RESPONSE",
  "requestId": "<REQUEST_UUID>",
  "deviceId": "<DEVICE_ID>",
  "success": true,
  "data": [{},null,null]
}

    Parameters

    Parameter Description
    REQUEST_UUID A UUID to differentiate this request from others.
    COMMAND One of the Commands below
    DEVICE_ID A unique identifer for the specific oven. In order to find the device ids that are available to you, you can wait for an Oven State message to be received.

    Send Command Payloads

    Start Cook

    {
    "id": "<COMMAND_UUID>",
    "type": "startCook",
    "payload": {
        "cookId": "<DEVICE_ID>",
        "stages": [
            {
                "stepType": "stage",
                "id": "<STAGE_UUID>",
                "title": "",
                "description": "",
                "type": "preheat",
                "userActionRequired": false,
                "temperatureBulbs": {
                    "wet": {
                        "setpoint": {
                            "fahrenheit": 131,
                            "celsius": 55
                        }
                    },
                    "mode": "wet"
                },
                "heatingElements": {
                    "top": {
                        "on": false
                    },
                    "bottom": {
                        "on": false
                    },
                    "rear": {
                        "on": true
                    }
                },
                "fan": {
                    "speed": 100
                },
                "vent": {
                    "open": false
                },
                "rackPosition": 3,
                "steamGenerators": {
                    "relativeHumidity": {
                        "setpoint": 100
                    },
                    "mode": "relative-humidity"
                }
            }
        ]
    }
}

    This must be sent within a Send command.

    Parameter Description
    COMMAND_UUID A UUID to differentiate this command from others.
    DEVICE_ID A unique identifer for the specific oven. In order to find the device ids that are available to you, you can wait for an Oven State message to be received.
    STAGE_UUID A unique identifer to differentiate this cooking stage from others.

    Stop Cook

    {
    "id": "<UUID>",
    "type": "stopCook"
}

    This must be sent within a Send command.

    Parameter Description
    UUID A UUID to differentiate this command from others.
